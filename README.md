# 🐉 Dragon Shop Bot

ربات فروش دیجیتال تلگرامی با کیف‌پول داخلی، پنل مدیریت کامل (CRUD دینامیک)، سیستم زیرمجموعه‌گیری و شارژ کارت‌به‌کارت/کریپتو.

## امکانات

- احراز هویت اجباری با شماره تلفن ایرانی (دکمه‌ی رسمی تلگرام "ارسال شماره تلفن")، فقط شماره‌های 09xxxxxxxxx پذیرفته می‌شن
- دکمه‌ی «⬅️ بازگشت» در تمام مراحل ورودی متنی (خرید، شارژ کیف پول) برای انصراف راحت کاربر
- کیف پول داخلی: کاربر شارژ می‌کنه، بعد خریدها آنی از موجودی کسر میشه
- دسته/محصول کاملاً داینامیک: ادمین از داخل ربات دسته و محصول اضافه/ویرایش/غیرفعال/حذف می‌کنه، بدون نیاز به تغییر کد
- برای هر محصول قابل تنظیمه که آیا نیاز به گرفتن اطلاعات از کاربر داره (مثلاً یوزرنیم اکانت مقصد) و متن سوالش هم سفارشیه
- مدیریت استوک (کد/اکانت) برای هر محصول، تحویل خودکار در صورت موجود بودن
- دسته‌ی «خرید ارز / کریپتو» با نرخ قابل‌تنظیم و محاسبه‌ی خودکار قیمت
- تایید/رد دستی رسید شارژ کیف پول (کارت‌به‌کارت یا کریپتو) توسط ادمین
- سیستم زیرمجموعه‌گیری با درصد پورسانت قابل تنظیم
- چند ادمین با سطح دسترسی، همه از داخل ربات مدیریت می‌شن
- ویرایش برند (نام فروشگاه، متن خوش‌آمدگویی، آیدی پشتیبانی) از پنل تنظیمات
- آمار فروش، پرفروش‌ترین محصولات، ارسال پیام همگانی
- قابلیت فعال‌سازی عضویت اجباری در کانال
- شماره مجازی به‌صورت دسته‌بندی آماده ولی غیرفعال اضافه شده (هروقت خواستی از پنل ادمین فعالش کن)

## نصب سریع (لوکال)

```bash
pip install -r requirements.txt
cp .env.example .env   # مقادیر رو پر کن
python bot.py
```

- `BOT_TOKEN` رو از @BotFather بگیر
- `OWNER_ID` آیدی عددی خودته؛ از @userinfobot بگیر

## دیپلوی روی Railway (پلن رایگان ۳۰ روزه)

### ⚠️ نکته‌ی مهم درباره دیتابیس

Railway به‌صورت پیش‌فرض فضای دیسک رو بین دیپلوی‌ها/ری‌استارت‌ها **پاک می‌کنه** (فایل‌سیستم موقته). اگه بدون Volume دیپلوی کنی، با هر ری‌استارت کل دیتابیس (کاربرها، موجودی کیف‌پول، سفارشات) از بین می‌ره. برای جلوگیری از این اتفاق **حتماً یک Volume بساز**:

1. داخل پروژه‌ی Railway برو به تب **Volumes**
2. یک Volume جدید بساز و Mount Path رو بذار روی: `/data`
3. متغیر محیطی `DB_PATH` رو دقیقاً همون‌طور که در `.env.example` هست، `/data/dragonshop.db` بذار (پیش‌فرض کد هم همینه)

### مراحل دیپلوی

1. این پوشه رو به یک ریپازیتوری گیت‌هاب پوش کن (یا مستقیم از طریق Railway CLI آپلود کن)
2. در Railway: **New Project → Deploy from GitHub repo**
3. در تب **Variables** این‌ها رو اضافه کن:
   - `BOT_TOKEN`
   - `OWNER_ID`
   - `DB_PATH=/data/dragonshop.db`
4. طبق بالا یک Volume با مسیر `/data` بساز و به سرویس وصلش کن
5. Railway به‌صورت خودکار `requirements.txt` و `Procfile` رو تشخیص می‌ده و ربات رو اجرا می‌کنه (پروسه‌ی `worker`)
6. توی تب **Deployments → Logs** چک کن که پیام polling شروع شده باشه و خطا نداشته باشه

بعد از این ۳۰ روز رایگان تموم بشه، برای ادامه‌ی کار باید یک پلن پولی (Hobby) رو فعال کنی وگرنه ربات خاموش می‌شه.

## اولین قدم‌ها بعد از بالا اومدن ربات

1. با اکانت خودت (همون `OWNER_ID`) به ربات پیام `/start` بده
2. `/admin` رو بزن تا پنل مدیریت باز بشه
3. از «⚙️ تنظیمات پرداخت» شماره کارت، نام صاحب کارت و/یا آدرس کیف‌پول کریپتو رو ست کن
4. از «📂 مدیریت دسته‌ها» وارد هر دسته بشو و محصولاتش (مثلاً پرمیوم ۳/۶/۱۲ ماهه، بسته‌های استارز) رو با قیمت دلخواه بساز
5. برای محصولاتی که نیاز به استوک دارن (کد پرمیوم و غیره)، از «📦 افزودن استوک» کدها رو وارد کن
6. دسته‌ی «شماره مجازی» طبق خواسته‌ات غیرفعاله؛ هروقت آماده بود از پنل فعالش کن

## ساختار پروژه

```
bot.py            نقطه‌ی ورود و اجرای ربات
config.py         تنظیمات و متغیرهای محیطی
database.py       لایه‌ی دیتابیس (SQLite) و تمام کوئری‌ها
keyboards.py      کیبوردهای شیشه‌ای و معمولی
states.py         استیت‌های FSM
utils.py          توابع کمکی مشترک
handlers/user.py  فلوی کاربر عادی (خرید، کیف‌پول، رفرال)
handlers/admin.py پنل مدیریت کامل
```

## توسعه‌ی بعدی پیشنهادی

- اتصال خودکار به تامین‌کننده‌ی استارز/پرمیوم به‌جای وارد کردن دستی استوک (وقتی آماده بودی بگو کمکت کنم به API متصلش کنم)
- درگاه پرداخت آنلاین (زرین‌پال و غیره) به‌عنوان روش سوم شارژ کیف‌پول
- گزارش‌گیری Excel از سفارشات
